HOW THE SOLVER THINKS

Each turn, the solver runs through a decision cascade, stopping as soon as it finds a move:

1. Opening move — always reveals a corner to maximize the initial flood-fill.

2. Single-cell deductions — for each revealed number, if hidden neighbors equals remaining mines they're all mines; if remaining is zero they're all safe.

3. Constraint subset reasoning — builds a constraint per frontier number ("exactly k of these cells are mines"), then compares pairs. If one is a subset of another, the difference cells may be deterministic.

4. Global mine count — if total remaining mines equals total hidden cells, all are mines. If zero remain, all are safe.

5. Constraint enumeration — groups frontier cells into connected components, enumerates every valid mine assignment via backtracking, and weights combinations by the global mine count. Produces exact per-cell probabilities. Cells at 0% or 100% are handled deterministically; otherwise reveals the safest cell, preferring the frontier for information gain.
